## 🆕 Added

### modules/alb (신규 모듈)

| 리소스 | 이름 | 상세 |
|--------|------|------|
| Application Load Balancer | `everybuddy-alb` | internet-facing, AZ-a + AZ-b |
| Security Group | `everybuddy-alb-sg` | 80/443 전체 허용 |
| Target Group | `everybuddy-backend-tg` | HTTP:8080, health check `/actuator/health` |
| Listener (80) | - | HTTP → HTTPS 301 리다이렉트 |
| Listener (443) | - | HTTPS → Target Group 포워딩 |
| ACM Certificate | `everybuddy.cloud` | DNS 검증, ISSUED |
| Route53 A Record | `api.everybuddy.cloud` | ALB Alias 레코드 |

## 트래픽 흐름

```
클라이언트
  │
  │ HTTPS (api.everybuddy.cloud)
  ▼
Route53 → ALB everybuddy-alb
  │
  │ HTTP:8080
  ▼
Target Group everybuddy-backend-tg
  │
  ▼
EC2 everybuddy-backend (:8080 Spring Boot)
```

## 설계 결정

- ALB를 AZ-a (`public-backend`)와 AZ-b (`public-b`) 두 서브넷에 배치해 고가용성 확보
- HTTP(80)는 HTTPS로 강제 리다이렉트 — 평문 통신 차단
- ACM 인증서 DNS 검증 방식 사용 (자동 갱신 지원)
