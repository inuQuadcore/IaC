# v2.4.0 — 백엔드 서버 Private 전환

**날짜:** 2026-03-11
**커밋:** `c8d4175` Refector: public -> private 서버 이전

---

## 🆕 Added

- **EC2** `everybuddy-private-backend` — t3.small, Private Subnet (10.0.11.0/24, AZ-a)
- **Security Group** `everybuddy-private-backend-sg`
  - Inbound: 8080 — ALB SG만 허용
  - Inbound: 22 — Bastion SG만 허용
  - Outbound: 전체 허용 (NAT GW 경유)

## 🗑️ Removed

- 기존 Public Backend EC2 (`everybuddy-backend`, 18.136.148.40) 제거
- 기존 `everybuddy-backend-sg`의 직접 8080 오픈 규칙 제거

---

## ✏️ Changed

- ALB Target Group → 새 Private Backend EC2 (private-app-a:8080) 로 타겟 변경
- 아웃바운드 인터넷 (DockerHub, S3, Firebase) → NAT GW 경유로 전환

---

## 변경 이유

Public IP를 가진 EC2를 인터넷에 직접 노출하지 않고 ALB 뒤에 배치해 공격 면 최소화.
외부에서 백엔드에 직접 접근하는 경로를 차단하고 모든 트래픽을 ALB를 통해서만 수신.

---

## 아웃바운드 의존성 (NAT GW 경유)

| 목적지 | 용도 |
|--------|------|
| Docker Hub | 이미지 pull |
| AWS S3 (ap-northeast-2) | 파일 업로드/다운로드 |
| Firebase Realtime DB | 채팅 실시간 동기화 |

---

## 최종 아키텍처

```
인터넷
  │
  ▼
ALB everybuddy-alb (internet-facing)
  │
  │ HTTP:8080 (ALB SG만 허용)
  ▼
Private App Subnet 10.0.11.0/24 (AZ-a)
  └── EC2 everybuddy-private-backend (t3.small)
        │
        │ SSH (Bastion SG만 허용)
        ▲
Bastion (public-b) ← GitHub Actions / 개발자

Private Backend 아웃바운드:
  └── NAT GW → 인터넷
```
