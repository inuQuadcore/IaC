## 🆕 Added

### modules/waf — 신규 모듈

AWS WAFv2 Regional을 ALB에 연결하는 모듈 추가.

**적용된 Managed Rule Group (우선순위 순):**

| Priority | Rule Group | 차단 대상 |
|----------|-----------|-----------|
| 1 | `AWSManagedRulesAmazonIpReputationList` | 봇, 스캐너, 악성 IP |
| 2 | `AWSManagedRulesCommonRuleSet` | OWASP Top 10 |
| 3 | `AWSManagedRulesKnownBadInputsRuleSet` | PHP, Log4j, ThinkPHP 등 |

**생성 리소스:**
- `aws_wafv2_web_acl` `everybuddy-waf` — REGIONAL scope, default action: allow
- `aws_wafv2_web_acl_association` — WAF → ALB 연결

**모듈 입력값:**

| 변수 | 설명 |
|------|------|
| `project_name` | 리소스 명칭 prefix |
| `alb_arn` | 연결할 ALB ARN |

## ✏️ Changed

### main.tf — waf 모듈 호출 추가

```hcl
module "waf" {
  source       = "./modules/waf"
  project_name = var.project_name
  alb_arn      = module.alb.alb_arn
}
```

---

## 아키텍처 변화

```
Internet
   │
  ALB  ← WAF (IpReputation + CommonRuleSet + KnownBadInputs)
   │
Private Backend EC2
```

WAF association 생성 즉시 ALB를 통과하는 모든 트래픽에 룰이 적용됨.

---

## 운영 참고

- CloudWatch Metrics로 각 룰별 차단/허용 수치 확인 가능
- `AWSManagedRulesCommonRuleSet`은 false positive 가능성 있음 (multipart 업로드, 특수문자 포함 body 등)
- 차단 이벤트 발생 시 해당 룰만 `COUNT` 모드로 전환 후 로그 분석 권장
